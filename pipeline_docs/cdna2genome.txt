Note
----
The script reads in each cDNA and clips it using Bio::EnsEMBL::Analysis::Tools::PolyAClipping. It then looks for the cDNA in Mole and finds its coordinates. If the coordinates are not found or can't be parsed, a message is sent to STDERR and the cDNA is not written to the outfile. If the cDNA is found, we check that it has not been clipped into the CDS and then write the cDNA's fasta to outfile and its annotation to annotation_file. This annotation file contains additional information for the use to know what has happened during clipping. In order for this annotation file to be used by exonerate's cdna2genome model, the annotation file with have to be edited to contain only the first 4 columns of each line
eg. less /lustre/scratch1/ensembl/user/targetted/cdna2genome/cdna.annot | awk '{print $1"\t"$2"\t"$3"\t"$4}' > /lustre/scratch1/ensembl/user/targetted/cdna2genome/annotation.txt

1.10 BestTargetted
==
You're probably running cdna2genome as part of an automatic gene annotation pipeline. If you're using cdna2genome as well as genewise to build targetted models, and then using the BestTargetted code to decide which models you prefer, remember that you will need to use some trickery to get BestTargetted to work with cdna2genome models. This is because the transcript_supporting_features for cdna2genome models are stored in the dna_align_feature table. (Targetted models from genewise have their supporting features stored in the protein_align_feature table.)

Without the trickery, you might get a silent error (does not throw), something like:
  Can't call method "seq" on an undefined value at /path/to/ensembl-analysis/modules/Bio/EnsEMBL/Analysis/Runnable/BestTargetted.pm line 191.

This error comes from cdna2genome models, because their supporting evidence is mRNA and it's not found in your protein index (/path/to/sequence/proteome/species.fa). You need to add these to your protein index.

You now have 2 options:
(i) Option 1:
Write a script eg. get_protein_seq_for_cdna.pl
that has your cDNA accession as the header but its matching protein sequence as the actual sequence.
eg: >NM_001075425.1|NP_001068893
MTPLLFPLLLASLLPSSSCNKANKHKPWIEAEYQGIVMENDNTVLLNPPLFALDKDAPLR
YAGEICGFRLHGSGVPFEAVILDKATGEGLIRAKEPVDCEAQKEHTFTIQAYDCGEGPDG
ANTKKSHKATVHVRVNDVNEFAPVFVERLYRAAVTEGKLYDRILRVEAIDGDCSPQYSQI

(ii) Option 2:
You could also do this with redundant sequence, where the same protein sequence is found twice in your file; once with the protein accession and once with the cDNA accession as a header. (This way might be easier)
eg
>NM_001075425.1
MTPLLFPLLLASLLPSSSCNKANKHKPWIEAEYQGIVMENDNTVLLNPPLFALDKDAPLR
YAGEICGFRLHGSGVPFEAVILDKATGEGLIRAKEPVDCEAQKEHTFTIQAYDCGEGPDG
ANTKKSHKATVHVRVNDVNEFAPVFVERLYRAAVTEGKLYDRILRVEAIDGDCSPQYSQI
>NP_001068893
MTPLLFPLLLASLLPSSSCNKANKHKPWIEAEYQGIVMENDNTVLLNPPLFALDKDAPLR
YAGEICGFRLHGSGVPFEAVILDKATGEGLIRAKEPVDCEAQKEHTFTIQAYDCGEGPDG
ANTKKSHKATVHVRVNDVNEFAPVFVERLYRAAVTEGKLYDRILRVEAIDGDCSPQYSQI

Depending on which option you chose, you will use a different method to create an index

For option (i), do:
  cd /path/to/sequence/proteome/
  # you have a species.fa file with all protein sequences here
  /software/ensembl/bin/indicate -d /path/to/sequence/proteome/ -f species.fa -i index -p singleWordParser
(You may need to hack indicate)

Now need to change this index that the files are pointing to:
Add the keep_single_analysis option
  vi /Bio/EnsEMBL/Analysis/Config/GeneBuild/BestTargetted.pm


For option (ii), do what is recommended in the normal genebuild documentation.

Good luck!
